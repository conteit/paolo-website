# Major Dependency Upgrades & Renovate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade 5 deferred major-version packages across 3 PRs ordered by risk, then replace Dependabot with Renovate for automated zero-touch maintenance.

- [ ] **Step 1: Create branch from main**


- [ ] **Step 2: Install TypeScript 6**

```bash

```

Expected: no errors.

- [ ] **Step 3: Verify package.json**

```bash
grep '"typescript"' package.json
```

Expected: `"typescript": "^6.0.3"`.

- [ ] **Step 4: Run typecheck and capture all errors**

```bash
npm run typecheck > /tmp/tc-c.log 2>&1; echo "Exit: $?"; cat /tmp/tc-c.log
```

If `Exit: 0` — done, skip to Step 6.

If errors appear, read them all before fixing anything. Common TypeScript 6 errors in this codebase:

- **`isolatedModules` stricter enforcement** — a file re-exports a type without `export type`. Fix: change `export { Foo }` to `export type { Foo }` for type-only exports.
- **`React` namespace not in scope** — TS6 tightens JSX type inference. Fix: add `import type React from 'react'` to the affected file, or change `React.ReactNode` return types to `ReactNode` with a named import.
- **Unused `@ts-ignore` comments** — TS6 errors on `@ts-ignore` that suppresses nothing. Fix: remove the comment.

Fix each error with the minimum change. Do not add `// @ts-ignore` or `any` to suppress — resolve the underlying issue.

- [ ] **Step 5: Re-run typecheck until clean**

```bash
npm run typecheck > /tmp/tc-c.log 2>&1; echo "Exit: $?"; tail -5 /tmp/tc-c.log
```

Expected: `Exit: 0`. Repeat fix → typecheck cycle until clean.

- [ ] **Step 6: Run build**

```bash
npm run build > /tmp/build-c.log 2>&1; echo "Exit: $?"; tail -5 /tmp/build-c.log
```

Expected: `Exit: 0`.

## Task D: Replace Dependabot with Renovate

**Branch:** `50-ci-renovate`

> Can be done in parallel with Tasks A/B/C — it only touches CI config files.

**Files:**
- Delete: `.github/dependabot.yml`
- Create: `renovate.json`

- [ ] **Step 1: Create branch from main**

```bash
git checkout main && git pull origin main
git checkout -b 50-ci-renovate
```

- [ ] **Step 2: Delete dependabot config**

```bash
git rm .github/dependabot.yml
```

- [ ] **Step 3: Create renovate.json**

Create `/Users/conteit/Code/GitHub/paolo-website/renovate.json` with this exact content:

```json
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": ["config:recommended"],
  "schedule": ["before 9am on Monday"],
  "labels": ["dependencies"],
  "packageRules": [
    {
      "groupName": "react-router",
      "matchPackageNames": ["react-router", "@react-router/**"]
    },
    {
      "groupName": "tailwindcss",
      "matchPackageNames": ["tailwindcss", "@tailwindcss/**"]
    },
    {
      "groupName": "react",
      "matchPackageNames": ["react", "react-dom", "@types/react", "@types/react-dom"]
    },
    {
      "groupName": "vite",
      "matchPackageNames": ["vite", "vite-tsconfig-paths", "@vitejs/**"]
    },
    {
      "matchUpdateTypes": ["patch", "minor"],
      "automerge": true,
      "automergeType": "pr",
      "requiredStatusChecks": ["typecheck", "build"]
    },
    {
      "matchUpdateTypes": ["major"],
      "automerge": false
    }
  ],
  "vulnerabilityAlerts": {
    "enabled": true,
    "labels": ["security"],
    "automerge": true
  },
  "dependencyDashboard": true
}
```

- [ ] **Step 4: Validate renovate.json is valid JSON**

```bash
node -e "JSON.parse(require('fs').readFileSync('renovate.json','utf8')); console.log('valid JSON')"
```

Expected: `valid JSON`.

- [ ] **Step 5: Commit**

```bash
git add renovate.json
git commit -m "ci: replace Dependabot with Renovate

- Package grouping (react-router, tailwindcss, react, vite families)
- Auto-merge patch/minor once typecheck + build pass
- Major updates always require human review
- CVE-only vulnerability alerts with auto-merge
- Dependency Dashboard issue for live update overview"
```

- [ ] **Step 6: Push and open PR**

```bash
git push origin 50-ci-renovate
```

Open PR targeting `main`. Title: `ci: replace Dependabot with Renovate`.

- [ ] **Step 7: Install Renovate GitHub App (manual — cannot be scripted)**

After the PR is merged:

1. Go to https://github.com/apps/renovate
2. Click **Install**
3. Select the `conteit/paolo-website` repository
4. Confirm installation

Renovate will open an onboarding PR within minutes of installation. Review and merge it — this activates Renovate on the repo and creates the Dependency Dashboard issue.

- [ ] **Step 8: Verify Dependency Dashboard appears**

After merging Renovate's onboarding PR, check the repository's Issues tab. A pinned issue titled **"Dependency Dashboard"** should appear showing all pending updates grouped by the rules in `renovate.json`.
